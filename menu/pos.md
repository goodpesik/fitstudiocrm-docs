# 3.9 Каса

> Сторінка **Каса** — основний робочий розділ касира для перегляду транзакцій, керування готівкою у касі та фіскальних звітів.

![](../_media/pos-overview.png ':size=400x241')

## 3.9.1 Верхня панель

> Над таблицями розташована верхня панель з основними діями та станом готівки.

**Зліва — фільтр і фіскальні звіти:**

- **Вибір періоду** — date range picker. Усі таблиці (Транзакції, Баланс готівки, Розширений звіт) рефільтруються за обраним діапазоном.
- **X Звіт** — миттєвий фіскальний звіт без закриття зміни (доступний коли зміна відкрита).
- **Звіт за день** — підсумковий фіскальний звіт за поточну зміну.

**Справа — каси та операції:**

- **Список акаунтів готівки** — кожен показує назву і поточний залишок. Дефолтний акаунт іменується **Баланс готівки**, інші — за заданою назвою.
- **+ Додати акаунт** — створює новий іменований акаунт готівки (див. [3.9.2 Акаунти готівки](#_392-Акаунти-готівки)).
- **Операції з готівкою** — відкриває модальне вікно для внесення / інкасації / витрати / переказу (див. [3.9.3 Операції з готівкою](#_393-Операції-з-готівкою)).

---

## 3.9.2 Акаунти готівки

> Студія може мати **один Дефолтний акаунт** і будь-яку кількість **іменованих акаунтів** (наприклад "Конверт оренди", "Зарплати"). Фізично гроші лежать в одній касі — акаунти існують для обліку призначення коштів.

**Правила роботи:**

- **Дефолтний акаунт** створюється автоматично, видалити його не можна.
- **Усі продажі (готівка/картка/оплата) завжди йдуть на Дефолтний акаунт.** Іменовані акаунти не приймають продажі напряму.
- **Переказ в іменований акаунт** — здійснюється через **[Операції з готівкою](#_393-Операції-з-готівкою) → Переказ** з Дефолтного.
- **Перекласти в касу (зворотний переказ)** — для кожного іменованого акаунту доступна кнопка 🔘, яка переказує всю/частину суми назад на Дефолтний.
- **Видалити акаунт** — кнопка 🔘. Якщо на акаунті є залишок, він автоматично переказується на Дефолтний перед видаленням.

**Створення нового акаунта:**

1. Натиснути **+** в зоні акаунтів.
2. Ввести назву, натиснути 🔘 для збереження або 🔘 для скасування.

## 3.9.3 Операції з готівкою

> Кнопка **Операції з готівкою** відкриває модальне вікно з вибором типу операції.

![](../_media/pos-cash-ops-modal.png ':size=300x181')

**Доступні операції:**

| Операція | Опис |
|---|---|
| **Внесення** | Поповнення Дефолтного акаунту готівкою (на початку зміни тощо). |
| **Інкасація** | Зняття готівки з Дефолтного акаунту (наприкінці зміни, передача в банк). |
| **Витрата** | Списання з Дефотного акаунту під конкретну витрату (опційно з категорією). |
| **Переказ** | Переклад суми з Дефолтного акаунту на один з іменованих акаунтів. |

**Поля:**

- **Сума** — обовʼязкове поле.
- **Акаунт призначення** — зʼявляється для **Переказ** (вибір з іменованих акаунтів).
- **Коментар** — текстове поле, додається до запису у Балансі готівки.

Збереження операції створює відповідні записи у вкладці **Баланс готівки** з категоріями `DEPOSIT`, `WITHDRAW`, `EXPENSE`, `TRANSFER_OUT` / `TRANSFER_IN`.

## 3.9.4 Транзакції

> Вкладка **Транзакції** показує всі продажі за обраний період.

![](../_media/pos-transactions.png ':size=420x253')

**Колонки:**

- **Дата** — дата і час транзакції.
- **Сплачено** — сума у грн; для повернень префікс `-`.
- **Метод оплати** — бейдж:
  - **Особовий рахунок** (якщо транзакція оплачена з гаманця клієнта).
  - **Готівка / Картка / Безготівкова** — інакше, відповідно до способу.
- **Тип** — бейдж **Продаж** / **Повернення**.
- **Клієнт** — імʼя клієнта (для продажу сертифіката тут показується **Сертифікат**).
- **Створено** — імʼя користувача, який провів транзакцію.
- **Статус платежу** — **Підтверджено** / **Очікує підтвердження** (для безготівкових при ПРРО з ще не схваленим платежем).
- **Дії**:
  - 🔘 **Повернення** — відкриває [модальне вікно повернення](#_3941-Повернення).
  - 🔘 **Змінити метод оплати** — для виправлення помилки касира одразу після продажу.

### 3.9.4.1 Повернення

> Модальне вікно повернення дозволяє:

- **Повне повернення** транзакції — повертає всі позиції клієнту.
- **Часткове повернення** — оберіть конкретні позиції (абонементи / продукти) до повернення.
- **Метод повернення** автоматично відповідає методу оригіналу:
  - Готівка → з Дефолтного акаунту;
  - Картка → відмічається у фіскальному запису;
  - Особовий рахунок → сума повертається на [Особовий рахунок клієнта](/menu/clients#_3211-Особовий-рахунок).

> ⚠️ Якщо тренування з абонементу вже використано — повернення створить компенсаційний запис у [Історії абонементу](/menu/clients#_325-Історія).

## 3.9.5 Баланс готівки

> Вкладка **Баланс готівки** показує всі рухи коштів по всіх акаунтах.

![](../_media/pos-cash-balance.png ':size=420x253')

**Колонки:**

- **Дата / Час**
- **Акаунт** — Дефолтний або назва іменованого
- **Тип** — DEPOSIT / WITHDRAW / EXPENSE / TRANSFER_OUT / TRANSFER_IN / RESET
- **Сума** (зі знаком: `+` для надходження, `–` для списання)
- **Залишок** — після операції
- **Створив** — імʼя касира
- **Коментар**

Усі рухи відсортовані за датою (за замовчуванням — спадання).

---

## 3.9.6 Розширений Звіт

> Вкладка **Розширений Звіт** — детальний фінансовий звіт за обраний період (FC-268).

![](../_media/pos-advanced-report.png ':size=420x253')

**Включає:**

- **Підсумок по методах оплати** (готівка/картка/особовий рахунок/сертифікат)
- **Підсумок по типах продажу** (абонементи / продукти / поповнення особового рахунку)
- **Підсумок по категоріях витрат**

Звіт автоматично виключає рядки з нульовими сумами.
