# 3.1 Розклад

> Вигляд вкладки **Розклад**

![](../_media/schedule-main.png ':size=190x100')

> Розклад має вигляд календаря. За замовчуванням відображається поточний тиждень.

**Елементи:**

- **Date picker** — дозволяє вибрати конкретну дату. Після вибору буде показано тиждень, який містить обрану дату.
- Зліва та справа від date picker розташовані кнопки назад «<<» та вперед «>>», які дозволяють здійснювати перехід по тижнях.
- Кнопка **Фільтр**

![](../_media/scedule-filter.png ':size=190x100')

  - Дозволяє фільтрувати відображення елементів у розкладі.
  - Фільтр по залах.
  - Фільтр по тренерах.
  - Фільтр по типах тренування (Регулярне, Персональне, Оренда).

- Кнопка **Експорт у PDF** — після натискання генерує PDF-документ з елементами, які відображені у календарі з урахуванням вибраних фільтрів.

> Картка тренування містить:

- Назву тренування.
- Час тренування.
- Імʼя тренера або **Оренда**.

> При наведенні на будь-яку клітину (порожню або з тренуванням, якщо клітина знаходиться в майбутньому часі або належить до поточного дня)  
> зʼявляється кнопка, яка дозволяє додати тренування.

![](../_media/add-workout.png ':size=150x100')

[Подробніше про створення тренування](menu/workouts?id=_331-Створити-тренування)

> Після натискання на картку тренування показується модальне вікно:

![](../_media/schedule-card-details.png ':size=100x100')

**Модальне вікно містить:**

- Назву тренування.
- Тип тренування: групове, персональне
- Аватар та імʼя тренера (аватар показується, якщо він заданий на сторінці тренера).
- Дату та час тренування.
- **Запланували відвідати** — кількість клієнтів, які записалися на це тренування.
- **Місць лишилося** — показується, якщо для тренування задана максимальна кількість учасників.  
  Розраховується за формулою:  
  ```Максимальна кількість учасників – Кількість записаних клієнтів```
- **Зареєстровано** — кількість клієнтів, яких уже зареєстровано на тренування.
- **Відмінено** - кількість клієнтів, які відмінили тренування.
- **Резерв місць** - доступно за умови резерву.
- Кнопка **Відмінити тренування** — буде показано [модальне вікно "Ви впевнені, що хочете продовжити?](../_modals/are-you-sure-modal.md ':include'). Якщо підтвердити, тренування буде відмінено на цей день.
- Тогл **Записати клієнта**
- Тогл **Кнопка налаштувань**

**Записати клієнта**

> Якщо активувати зʼявиться селект для вибору клієнта

![](../_media/schedule-card-details-2.png ':size=50x50')

> після вибору

![](../_media/new-client-1.png ':size=100x70')

- Тогл **Дублювати запис** - поставить запис на автоматичний повтор
- Кнопка  **Підтвердити запис**- після натискання клієнта буде записане на вибране тренування

> Також доступна опція **+ Новий клієнт**

![](../_media/add-client-select.png ':size=70x70')
![](../_media/new-client-2.png ':size=100x70')

> після вибору клієнт буде записаний на тренування

**Кнопка налаштувань**

![](../_media/schedule-card-details-3.png ':size=70x70')
- Кнопка відміни тренування
- Тоггл заміна тренера

![](../_media/substitute-2.png ':size=50x70')

  - Вибір тренера для заміни.

> Після вибору тренера для заміни стає доступною кнопка **Зберегти заміну**. Після натискання зміни зберігаються, картка закривається.

> Внести зміни повторно можна або відкривши картку ще раз, або відредагувавши тренування на сторінці тренувань.

> Коли заміна активна, буде доступна кнопка **Видалити заміну**


**Клієнти**

> Якщо на тренування записані клієнти їх імена будуть відображені на картці деталей тренування
> При натисканні на імʼя клієнта буде доступно контекстне меню (залежить від статуса клієнта та тренування)
> клієнти виводяться з урахуванням статусу, що позначено кольором

- **Білий** - Заплановано візит
- **Зелений** - Зареєстровано
- **Червоний** - Відмінено
- **Помаранчевий** - Резерв

**Контекстне меню**

- Для клієнтів в статусі - **Білий** - Відмінити тренування. Зареєструвати клієнта (Тількі якшо тренування відбувається сьогодні)
- Для клієнтів в статусі - **Зелений** - Відмінити реєстрацію. (Може відмінити помилкову реєстрацію. Поточний абонемент залишається без змін, але клієнту повертається 1 разове тренування яке може бути використано пізніше).
- Для клієнтів в статусі - **Червоний** - Змінити статус на відвідано - Дозволяє змінити колір на зелений. Ніякіх змін в абонементах не відбувається.
- Для клієнтів в статусі **Помаранчевий** - Відмінити резерв. Що відмінить тренування на цю дату для обраного клієнта

**Резерв місць**

Якщо тренування повторюється, та на нього записані клієнти в яких стоїть налаштування дублювання записів, для тренувань які відбудуться у майбутньому в такому разі будуть доступні резерв місць. Резерв місць показує клієнтів які потенційно буду записані на це тренування автоматично. Залишок місць для тренування буде показано з урахуванням кількості резерва для контролю овербукінгу.

## Вихідні студії

> Кнопка **Вихідні студії** у верхній панелі розкладу відкриває попап для керування днями коли студія не працює (свята, технічні перерви тощо).

### Як додати вихідний

1. Натиснути **Вихідні студії**.
2. Натиснути **Додати виключення** — зʼявиться рядок з date picker.
3. Обрати дату.
4. Можна додати кілька дат поспіль.
5. Натиснути **Зберегти**.

![](../_media/schedule-holidays-popover.png ':size=300x250')

### Що відбувається при збереженні

- Запускається фоновий job (показується [модалка фонових задач](/pages/processing#фонові-задачі)).
- Усі тренування на обраних датах **відміняються** автоматично; залежні бронювання клієнтів отримують статус **Відмінено**.
- Список вихідних зберігається у налаштуваннях студії.

### Як видалити вихідний

1. Відкрити попап **Вихідні студії**.
2. Натиснути 🔘 (іконка кошика) у рядку з потрібною датою.
3. Натиснути **Зберегти**.

> ⚠️ Видалення вихідного **відновлює** тренування і бронювання на цю дату — теж через фоновий job.

> ℹ️ Минулі дати у списку лишаються для історії, але редагувати їх не можна (`minDate` обмежує picker сьогоднішньою датою).
